# QuizForge

把包含**题目、答案、详解**的 Markdown，组织进**笔记本**，转成可交互的测验网站。界面采用 `DESIGN.md` 中的 Claude 暖色（cream/coral）设计系统，并支持深色主题。纯静态、无后端，所有数据存在浏览器本地。

技术栈：Vite + React。

## 功能

### 笔记本（Books）
- **笔记本式首页**：测验按笔记本分组展示（图标 + 配色，类 NotebookLM）。
- **内置笔记本**：`/content/<文件夹>/` 下的子目录即一本只读笔记本，随仓库发布。
- **自建笔记本**：在浏览器中「新建笔记本」，自定义名称 / 图标 / 配色，上传或粘贴 Markdown 添加测验，存于 localStorage。可编辑、删除笔记本与其中的测验。
- **可刷新的导航**：视图状态写进 URL hash（`#book=…`、`#play=…`），刷新后停留在当前页面、链接可收藏。

### 答题
- **单选 / 多选 / 判断题**：标记多个 `[x]` 即为多选题（需点击「提交」后判分）。
- **键盘操作**：数字键 `1‑9` / 字母键 `A‑Z` 选择选项，`Enter` 提交多选或进入下一题，方向键移动焦点。
- **即时反馈**：选错显示「Not quite」，并高亮正确项、展开逐项详解；`> Explain` 提供整题讲解。
- **LaTeX 公式**：`$...$` / `$$...$$`，由 KaTeX 渲染。
- **进度持久化**：答题进度自动存入 localStorage，重开同一测验自动续答；「重新开始」清空。
- **上一题 / 错题复盘**：可回看已答题目；结束后查看错题（正确答案 + 详解）。
- **只练错题**：结束后一键「重练错题」，仅用做错的题再来一轮。
- **乱序**：front-matter `shuffle: true` 打乱题序与选项顺序（同一次作答内顺序稳定）。

### 结束与导出
- **用时与成绩历史**：显示总用时、每题用时、对错，以及与上次/最佳成绩的进步趋势。
- **多格式导出**：把错题导出为 **Markdown**（可重新导入，含「你选择了…，正确答案为…」标注）、**Anki CSV** 或 **Quizlet** 文本。
- **分享链接**：把整份测验压缩进 URL（`#q=...`），分享即用，无需服务器。

### 其他
- **深色 / 浅色主题**：右上角切换，选择被记住。
- **移动端适配**：窄屏下单列布局、触摸友好。
- **可安装（PWA）**：支持离线访问，可安装到桌面 / 主屏。首页提供「📲 安装应用」按钮（仅在浏览器判定可安装时出现），也可用地址栏的安装图标。
- **可访问性**：选项为 ARIA radio/checkbox，支持方向键与回车/空格操作。

## 两种使用方式

1. **内置笔记本（构建时）**：在 `content/` 下按**文件夹**组织 `.md`，例如 `content/数据结构与算法/排序算法.md`。每个文件夹会成为一本只读笔记本，自动出现在首页（通过 `import.meta.glob('/content/**/*.md')` 打包，无需额外步骤）。直接放在 `content/` 根目录的文件归入「示例」笔记本。
2. **自建笔记本（运行时）**：在首页「新建笔记本」，进入后拖入 `.md` 文件或粘贴 Markdown 文本添加测验。笔记本与测验保存在浏览器 localStorage 中。

### URL / 路由

应用通过 hash 记录当前视图，刷新即停留：

| Hash | 视图 |
| --- | --- |
| `#` | 笔记本列表（首页） |
| `#book=<bookId>` | 某本笔记本 |
| `#play=<bookId>/<quizId>` | 正在作答的测验 |
| `#q=<compressed>` | 通过分享链接打开的测验 |

## Markdown 格式

```markdown
---
title: 排序算法测验
sources: 5            # 可选 → 显示 “View prompt and N sources”
---

## 题目写在二级标题里，可含行内公式 $O(n \log_2 n)$ 或块级公式 $$...$$

- [ ] 选项文本（错误）
  缩进在选项下方的这一行，是该选项的详解（答题后显示）。
- [x] 选项文本（正确）
  用 [x] 标记唯一正确答案。
- [ ] 又一个选项

> Explain: 可选。整道题的讲解，答题后点击 Explain 按钮展开。
```

### 规则

- 每道题以 `## ` 开头。
- 选项用任务列表语法：`- [ ]` 表示选项，`- [x]` 标记**唯一**正确答案。
- 选项标号 `A/B/C/D` 按顺序自动分配，无需手写（手写的 `A.` 前缀会被自动去除）。
- 紧跟在某个选项下方、**带缩进**的文字，是该选项的详解；可省略。
- `> Explain:` 开头的引用块，是整道题的讲解；可省略。
- 公式用 `$...$`（行内）和 `$$...$$`（块级），由 KaTeX 渲染；题干、选项、详解均支持 GFM Markdown。

格式不合法（无正确答案、多个正确答案、题目没有选项等）时，界面会给出明确的错误提示，而不是崩溃。

完整示例见 [`src/samples/example.md`](src/samples/example.md) 与 [`content/sorting.md`](content/sorting.md)。
