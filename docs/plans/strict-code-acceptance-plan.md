# QuizForge 严格代码验收计划

## 验收范围

本计划用于验收当前工作区 `E:\Claude Project\14-QuizForge` 的代码状态，包含已修改文件和未跟踪文件。验收对象以实际代码、测试、构建产物和浏览器行为为准，不以 README 或设计文档的描述自动判定通过。

重点覆盖：

- Vite + React 应用基础可用性。
- 题库、笔记本、答题、历史记录、SRS、收藏、错题本。
- 新增错题整理流程。
- 导入导出、分享链接、PWA 构建。
- 桌面端和移动端核心页面可用性。

## 一票否决项

任一项失败，验收结论必须为“不通过”：

1. `npm test` 失败。
2. `npm run build` 失败。
3. 构建产物启动后首页白屏。
4. 首页、题库详情页、答题页、错题页、统计页任一核心页面出现未捕获运行时错误。
5. 新增错题整理功能导致原题库、答题进度、历史记录、收藏或 SRS 数据丢失。
6. 删除错题记录时误删原题库或用户自建题库内容。
7. 导入无效数据、空数据、无错题数据时页面崩溃。
8. 关键用户流程只能通过刷新、手工改 localStorage 或开发者工具才能继续。

## 自动化验收

按顺序执行：

```powershell
npm test
npm run build
npm run preview
```

通过标准：

- 全部测试通过，不能跳过失败用例。
- 测试数量如果减少，必须说明减少原因；否则视为回归风险。
- 构建完成并生成 `dist/`。
- 预览服务可打开构建产物。
- 浏览器控制台无阻塞级错误。

建议新增或确认已有自动化覆盖：

1. `wrongBook.js`
   - 答错题写入错题本。
   - 答对题从错题本移除。
   - 错题按最近错误时间倒序展示。
   - 批量删除只删除错题记录，不影响原题。

2. `WrongOrganize.jsx`
   - 能展示错题列表。
   - 能按来源笔记本筛选。
   - 能按关键词搜索。
   - 能单选、全选当前结果、取消选择。
   - 无选中项时批量按钮不可执行。
   - 删除后列表刷新且选择状态清空。

3. 错题转题库
   - 选中错题导出的 Markdown 可被 `parseQuiz` 重新解析。
   - 生成的新笔记本包含选中题目。
   - 新笔记本进入详情页后可正常答题。
   - 题目、选项、正确答案、解析和来源信息不丢失。

4. 导入导出
   - 导出所有 `quizforge:*` 数据。
   - 导入合法备份后恢复对应数据。
   - 导入非法 JSON 返回失败，不抛出未捕获错误。
   - 导入不包含错题的数据时不会显示错误的错题入口。

## 人工功能验收

### 1. 首页与题库

步骤：

1. 清空站点数据后打开首页。
2. 确认内置 `content/` 题库按笔记本展示。
3. 新建一个用户笔记本。
4. 向用户笔记本添加一份合法 Markdown quiz。
5. 添加一份非法 Markdown quiz。

通过标准：

- 首页不白屏。
- 内置题库只读，自建题库可编辑。
- 合法 quiz 可保存并进入详情。
- 非法 quiz 显示错误，不污染已有题库。
- 刷新后用户笔记本仍存在。

### 2. 答题流程

步骤：

1. 打开一个包含单选题的 quiz。
2. 使用鼠标选择答案并提交。
3. 使用键盘 `1-9` 或字母选择答案。
4. 使用 `Enter` 进入下一题。
5. 完成 quiz，查看结果页。
6. 重新进入同一 quiz，验证历史记录和趋势。

通过标准：

- 单选、多选、判断题判分正确。
- 答错后显示正确答案和解析。
- 刷新页面后非临时 quiz 恢复进度。
- 完成后记录成绩、用时和历史趋势。
- 重新开始不会保留旧答案。

### 3. 错题本

步骤：

1. 完成一套 quiz，至少答错 2 题。
2. 回到首页，进入错题本。
3. 确认错题数量与答错数量一致。
4. 对其中一道错题答对。
5. 回到错题本查看数量变化。

通过标准：

- 答错题被记录。
- 答对后对应错题记录被移除。
- 错题本只包含错题，不包含答对题。
- 错题本为空时页面不崩溃。

### 4. 错题整理

步骤：

1. 进入错题整理页。
2. 使用来源笔记本筛选。
3. 使用关键词搜索。
4. 勾选单题。
5. 全选当前筛选结果。
6. 取消部分选择。
7. 删除选中错题。
8. 将选中错题加入收藏。
9. 导出选中错题为 Markdown。
10. 将选中错题生成新笔记本。

通过标准：

- 筛选和搜索结果正确。
- 全选只作用于当前筛选结果。
- 删除只删除错题记录，不删除原题库。
- 加入收藏后收藏复习页可看到对应题。
- 导出的 Markdown 可重新导入或被解析。
- 生成新笔记本后自动跳转到新笔记本详情。
- 新笔记本内题目可正常答题。
- 空结果、无选中项、取消确认不会造成状态错乱。

### 5. 收藏、统计、SRS

步骤：

1. 收藏一道题。
2. 进入收藏复习。
3. 完成一次答题。
4. 进入统计页。
5. 进入 SRS 复习入口。

通过标准：

- 收藏状态可切换且刷新后保留。
- 收藏复习只包含收藏题。
- 答题完成后统计页显示历史数据。
- SRS 卡片状态根据答对或答错更新。
- 今日待复习数量与实际状态一致。

### 6. 导入导出与分享

步骤：

1. 导出错题 Markdown。
2. 导出 Anki CSV。
3. 导出 Quizlet 文本。
4. 复制分享链接并在新标签打开。
5. 导出全量备份。
6. 清空站点数据后导入全量备份。

通过标准：

- Markdown、CSV、Quizlet 文本字段不乱序、不丢答案。
- 分享链接能打开临时 quiz。
- 临时 quiz 不污染原 quiz 进度。
- 全量备份恢复后，用户题库、错题、收藏、历史记录存在。
- 非法备份文件不会破坏已有数据。
