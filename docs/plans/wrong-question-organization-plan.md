# 导入错题后整理功能计划

## 目标

用户导入备份或积累错题后，可以把错题筛选、分组、清理，并一键整理成新的题库/笔记本，方便后续专项复习。

## MVP 范围

### 1. 定义错题整理入口

- 在首页“错题本”旁边增加“整理错题”入口。
- 入口只在错题数量大于 0 时可用。
- 点击后进入错题整理页面，而不是直接开始答题。

### 2. 做错题整理页面

- 展示所有错题列表。
- 每条错题展示：
  - 题目
  - 来源笔记本
  - 来源 quiz
  - 错误次数
  - 最近错误时间
- 支持勾选题目。
- 支持按来源笔记本筛选。
- 支持关键词搜索。
- 支持“全选当前结果”。

### 3. 加入基础整理动作

- 删除选中的错题记录。
- 把选中的错题加入收藏。
- 把选中的错题导出为 Markdown。
- 把选中的错题整理成一个新的自建笔记本。

### 4. 生成新笔记本

- 默认笔记本名：`错题整理 YYYY-MM-DD`。
- 默认 quiz 名：`整理错题`。
- 每道题保留：
  - 原题干
  - 选项
  - 正确答案
  - 题目讲解
- 额外追加来源信息，例如来自哪个笔记本和 quiz。
- 生成成功后跳转到新笔记本详情页。

### 5. 导入后的衔接

- 现有“导入备份”成功后，不只刷新页面。
- 如果导入数据里包含错题，提示用户：`已导入 X 道错题，是否立即整理？`
- 用户确认后直接进入“整理错题”页。
- 用户取消则保持当前导入流程。

### 6. 数据层最小改动

- 不新建复杂数据库。
- 复用现有 `wrongBook.js`、`books.js`、`dataIO.js`。
- 只增加必要 helper：
  - 获取错题详情列表
  - 批量移除错题
  - 将错题转换为 Markdown quiz
  - 从错题创建用户笔记本

### 7. 测试

- 测试错题转 Markdown 的结果正确。
- 测试从错题创建新笔记本。
- 测试批量删除不会影响原题库。
- 测试导入后能识别错题数量。

## 第一版不做

- 不做 AI 自动分类。
- 不做复杂标签系统。
- 不做云同步。
- 不做多级文件夹。
- 不做复杂合并逻辑。

## 成功标准

- 用户可以在导入备份后发现错题数量。
- 用户可以进入错题整理页面筛选和勾选错题。
- 用户可以把选中的错题整理成新的自建笔记本。
- 用户可以删除、收藏或导出选中的错题。
- 核心数据转换和批量操作有测试覆盖。
